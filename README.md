🛰️ Project pokerFace: Naming & Architecture Manifest

1. File Naming ConventionsTo ensure high "traceability," all files in the src/app and src/features directories must follow the Suffix Convention.CategorySuffixExampleInterfaces...Interfaces.tsmatchInterfaces.ts, profileInterfaces.tsTypes...Types.tsgameTypes.ts, uiTypes.tsRedux Slices...Slice.tsmatchSlice.ts, authSlice.tsRedux Selectors...Selectors.tsprofileSelectors.ts, matchSelectors.tsMiddleware...Middleware.tssocketMiddleware.ts, profileMiddleware.tsUtilities...Utils.tscardUtils.ts, xpMathUtils.ts
2. Directory StructureMaintain a strict separation between Global App Logic and Feature Logic.src/app/: Global blueprints./interfaces: Common interfaces used across multiple features./types: Union types, Enums, and literal types./hooks: Custom React hooks (e.g., useSound).src/features/: Domain-specific logic.Each folder (e.g., /match) should contain its own Slice and Selectors.
3. Coding StandardsInterfaces vs. TypesUse Interfaces for data models and objects (e.g., PlayerInterface). Interfaces are extendable and perform better in TypeScript's compiler for large objects.Use Types for Unions, Aliases, and Function signatures (e.g., type PlayerType = 'human' | 'npc').Data Nesting (The "Russian Doll" Rule)To keep the state organized, we separate "The Soul" from "The Body":profile: Persistent data (XP, Level, Plei, Achievements). Lives in the database.currentMatch: Volatile data (Current Hand, Chips, Fold status). Wiped after every game.
4. Redux ImplementationSelectors: Never access state directly in a component. Always use a memoized selector from a ...Selectors.ts file.Safety: Always provide fallback values in selectors (e.g., ?? 0 or ?? []) to prevent UI crashes during initialization.5. Asset ManagementVector Assets: All UI wireframes and neon silhouettes are designed in Adobe Illustrator.Naming: Assets should match their DeckStyleType or MatchLocationType string values exactly (e.g., arrowBolt.png).
